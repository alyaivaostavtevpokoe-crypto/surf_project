# Документ фичи: Разработка экрана оформления записи (Booking Flow)

**Цель:** Реализовать интерактивный интерфейс бронирования мест на выбранный заезд с конфигурацией параметров (места, экипировка) и валидацией ответов со стороны имитационного API.

## Требования:
- Вызов шторки бронирования (`ModalBottomSheet`) по тапу на доступную карточку расписания.
- Контроль выбора количества мест: инкремент заблокирован, если выбран максимум свободных мест из `SlotModel`.
- Свитч включения аренды инвентаря.
- Вызов метода `bookSlot()` в `MockApiService` с индикацией загрузки на кнопке.
- Сценарий Успеха: Перенаправление на полноэкранный `BookingSuccessScreen`.
- Сценарий Отказа: Отображение всплывающей ошибки (`SnackBar`) без закрытия шторки, чтобы пользователь мог изменить параметры.

## Что реализовано в коде:
1. Разработан `BookingViewModel`, управляющий реактивным состоянием формы бронирования и лимитирующий счетчик мест на основе `slot.remainingSeats`.
2. Написана логика симуляции бизнес-правил в `MockApiService` (запрос более 4 мест принудительно вызывает ошибку бизнес-валидации для теста ошибок).
3. Сверстаны `BookingBottomSheet` и `BookingSuccessScreen` с полной обработкой блокировок интерфейса во время выполнения сетевого запроса.
