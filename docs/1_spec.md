## 1. User Stories для MVP клиентского мобильного приложения

В рамках MVP выделены две ключевые роли: **Гость** (неавторизованный пользователь) и **Клиент** (авторизованный пользователь).

| ID | Роль | Формулировка (Я как... хочу... чтобы...) | Критерии приемки (Acceptance Criteria) |
| :--- | :--- | :--- | :--- |
| **US-01** | Гость / Клиент | Я как пользователь хочу видеть список доступных заездов (слотов) и информацию о тренерах (мастерах), чтобы выбрать подходящее время. | - Отображается список слотов на текущий день.<br>- Видно время, цену и количество свободных мест. |
| **US-02** | Клиент | Я как авторизованный клиент хочу забронировать выбранный слот, указав необходимость аренды инвентаря, чтобы закрепить место за собой. | - Возможность выбрать чекбокс «Аренда инвентаря».<br>- После подтверждения создается бронь в статусе "Создана". |
| **US-03** | Клиент | Я как клиент хочу видеть список моих текущих и прошедших бронирований, чтобы отслеживать свое расписание. | - Разделение на «Активные» и «Прошедшие».<br>- Отображается актуальный статус заезда. |
| **US-04** | Клиент | Я как клиент хочу видеть статус, если занятие отменено мастерской, чтобы вовремя скорректировать планы. | - Если у слота стоит флаг отмены, бронь визуально подсвечивается как «Отменена мастерской». |
