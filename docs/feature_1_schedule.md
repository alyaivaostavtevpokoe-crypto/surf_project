# Документ фичи: Разработка экрана «Расписание заездов»

**Цель:** Реализовать отображение актуального списка доступных для бронирования слотов с выводом всей информации о заезде, а также корректно обработать состояния загрузки, ошибок и отсутствия данных.

## Требования:
- Запрос данных через архитектурный компонент ViewModel из сервиса `MockApiService`.
- Отображение карточек занятий (дата, время, имя инструктора, его рейтинг, цена, потребность в экипировке и доступный остаток мест).
- Визуальное выделение отмененных мастерской слотов.
- Обработка сценария пустого списка (`Empty State`) с красивой заглушкой.

## Что реализовано в коде:
1. Создан `SlotsListViewModel` на базе `ChangeNotifier` (MVVM), который инкапсулирует стейт-машину экранов (`loading`, `empty`, `success`, `error`).
2. Сверстан UI-компонент карточки заезда `SlotCard` с условным рендерингом для заблокированных/отмененных мастерской слотов и индикацией заполненности мест.
3. Разработан экран `SlotsListScreen` с динамическим переключением состояний с помощью `AnimatedBuilder`, исключающий лишние перерисовки интерфейса.
