# Отчет о баге №2: Повторная отправка формы при асинхронном запросе (Race Condition / Double Submit)

**Статус:** Исправлен (Fixed)  
**Компонент:** `BookingViewModel`  
**Критичность:** High (Приводит к дублированию транзакций и некорректному списанию мест)  

## 1. Симптом (Как воспроизводился баг):
1. Пользователь открывает шторку бронирования `BookingBottomSheet` для доступного слота.
2. Пользователь совершает быстрое двойное нажатие (Double Tap) по кнопке «Записаться».
3. Из-за асинхронной задержки сети в `MockApiService` (1000 мс) интерфейс не успевает мгновенно заблокировать кнопку до прихода следующего кадра отрисовки.
4. В логах выполнения фиксируется два параллельных сетевых запроса к методу `bookSlot` с одинаковыми параметрами. Если бы это был реальный бэкенд, у пользователя списались бы деньги или забронировалось в 2 раза больше мест, чем планировалось.
